警惕拷贝构造函数

匿名对象一个析构；参数没有引用&，算一个；
(初始化与赋值是两个不同的概念)赋值=算一个，初始化不算；

深拷贝与浅拷贝

析构两次后C++编译器报错 堆malloc
如果不自定义拷贝函数，=这种默认的拷贝行为就是浅拷贝，共用同一地址，如若改变其中一个，其他值一起变。
这种将对象所持有的其它资源一并拷贝的行为叫做深拷贝，我们必须显式地定义拷贝构造函数才能达到深拷贝的目的。每个拷贝拥有独立的内存空间，防止析构报错。

注：如果一个类拥有指针类型的成员变量，那么绝大部分情况下就需要深拷贝，因为只有这样，才能将指针指向的内容再复制出一份来，让原有对象和新生对象相互独立，彼此之间不受影响。如果类的成员变量没有指针，一般浅拷贝足以。

结论：malloc不会调用类的构造函数,free不会调用类的析构函数

C++类对象中的成员变量和成员函数是分开存储的

一、成员变量(静态、非静态)

1普通成员变量：存储于对象中，与struct变量有相同的内存布局和字节对齐方式

2静态成员变量：存储于全局数据区中

二、成员函数(静态、非静态、虚拟)
存储于代码段中。

this指针隐含

void setxy(int a,int b ){X=a;Y=b;}
void setxy(Simple *const this ,int a,int b){this->X=0;this->Y=b;}

在 C++ 中，每一个对象都能通过 this 指针来访问自己的地址。this 指针是所有成员函数的隐含参数。因此，在成员函数内部，它可以用来指向调用对象。
友元函数没有 this 指针，因为友元不是类的成员。只有成员函数才有 this 指针。

友元函数/友元类

若B类是A类的友员类，则B类的所有成员函数都是A类的友员函数
友员类通常设计为一种对数据操作或类之间传递消息的辅助类

运算符重载

重载：函数名相同，函数的参数个数、参数类型或参数顺序三者中必须至少有一种不同。函数返回值的类型可以相同，也可以不相同。发生在一个类内部。多个同名函数同时存在，具有不同的参数个数/类型。重载是一个类中多态性的一种表现。

重写：也叫做覆盖，一般发生在子类和父类继承关系之间。子类重新定义父类中有相同名称和参数的虚函数。(override)

重定义：也叫做隐藏，子类重新定义父类中有相同名称的非虚函数 ( 参数列表可以不同 ) ，指派生类的函数屏蔽了与其同名的基类函数。可以理解成发生在继承中的重载。

虚继承二义性

1、一般：
c1.b = 500; //继承的二义性 和 虚继承解决方案
//c1.B1::b = 500;
//c1.B2::b = 500;
2、引用虚继承 virtual public 解决二义性

多态性vitrtual

多态成立的三个条件

//1 要有继承
//2 要有函数重写。。。C 虚函数
//3 要有父类指针（父类引用）指向子类对象

纯虚函数如何定义？使用时应注意什么？
答案：virtualvoid f()=0; 是接口，子类必须要实现

多态的实现效果？
多态：同样的调用语句有多种不同的表现形态；
多态实现的三个条件？
有继承、有virtual重写、有父类指针（引用）指向子类对象。

在什么情况下应当声明虚函数？
构造函数不能是虚函数。建立一个派生类对象时，必须从类层次的根开始，沿着继承路径逐个调用基类的构造函数。析构函数可以是虚的。虚析构函数用于指引 delete 运算符正确析构动态对象

a编译时多态性：通过重载函数实现 。
b运行时多态性：通过虚函数实现

函数的重载是在编译的时候发生，而虚函数是在运行的时候确定。其实有时候称函数的重载为静态多态性，而虚函数为动态多态性

继承的多态怎么实现?
多继承
尽管virtual是private的，子类中重写函数改为public,protected也可以，体现了多态。
指针大小，虚指针的大小?
vptr (类的第一个成员)->vtable 同类对象，共享一个vtable 对象里的虚表

一个具有纯虚函数的的类称为抽象类。

什么是动态多态？
由于有了虚函数，因此动态多态是在运行时完成的，也可以叫做运行期多态。

什么是静态多态？
在模板编程及泛型编程中，是以隐式接口和编译器多态来实现静态多态。

相同点：
都能够实现多态性，静态多态/编译期多态、动态多态/运行期多态；
都能够使接口和实现相分离，一个是模板定义接口，类型参数定义实现，一个是基类虚函数定义接口，继承类负责实现；
